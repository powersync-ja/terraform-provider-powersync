# Resource: powersync_project

Manages a PowerSync project. Creating a project mirrors the dashboard's create flow; most internal fields (template_id, source, features, vcs_mode) are hardcoded.

## Example Usage

```terraform
data "powersync_organization" "main" {
  id = "64b3f8e1a2c4d5e6f7080912"
}

resource "powersync_project" "main" {
  org_id = data.powersync_organization.main.id
  name   = "My Project"
  region = "eu"

  # Allow `terraform destroy` even when un-managed instances exist under this
  # project. Without this flag the destroy fails with a guard error.
  # force_destroy = true
}

output "project_id" {
  value = powersync_project.main.id
}
```

## Schema

### Required

- `name` (String) Human-readable project name. Only mutable field on update.
- `org_id` (String) Organization ID that will own the project. Changing this forces a new project.
- `region` (String) Default region for instances created under this project. One of: `eu`, `us`, `jp`, `au`, `br`. Surfaces as `default_region` on the `powersync_project` data source. Changing this forces a new project.

### Optional

- `force_destroy` (Boolean) When true, deleting this project will cascade-destroy any instances under it that are NOT managed by this Terraform configuration. Defaults to false; destroy is refused if non-tracked instances exist.

### Read-Only

- `id` (String) Project ID assigned by the API.
- `locked` (Boolean) Whether the project is locked.
- `trial` (Boolean) Whether the project is on a trial plan.
- `vcs_mode` (String) Version control mode for the sync config. Always `BASIC` today.

## Import

Import is supported using the following syntax:

```shell
# Import an existing project. The ID format is "org_id/project_id".
terraform import powersync_project.main 64b3f8e1a2c4d5e6f7080912/699ef9c371c56d0007320543
```
