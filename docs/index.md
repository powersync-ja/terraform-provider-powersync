# PowerSync Provider

The PowerSync provider lets you manage [PowerSync Cloud](https://www.powersync.com/) resources — organizations, projects, and instances — declaratively, the same way you manage the rest of your infrastructure in Terraform.

Typical use cases:

- Provision a PowerSync instance alongside the source database (RDS, Supabase, Neon, etc.) it replicates from, in a single `terraform apply`.
- Keep `dev`, `staging`, and `prod` instances in sync with their definitions in source control instead of clicking through a dashboard.
- Roll out sync rule changes through a pull request, with a Terraform plan as the review artifact.
- Recover an environment from scratch by re-applying the same Terraform configuration.

## Authentication

The provider authenticates against the PowerSync API using a **personal access token (PAT)**.

1. Generate a PAT in the PowerSync dashboard under [Account Settings → Personal Access Tokens](https://www.powersync.com/dashboard).
2. Export it as an environment variable before running Terraform:

   ```sh
   export PS_PAT_TOKEN="jpt_..."
   ```

The token is read automatically by the provider; you do not need to reference it in your `provider "powersync"` block. If you must pass it explicitly (e.g. inside a CI pipeline that already maps secrets into Terraform variables), use the `admin_token` argument — but prefer the environment variable to keep the token out of state files and plan output.

## Example Usage

```terraform
terraform {
  required_providers {
    powersync = {
      source  = "powersync-ja/powersync"
      version = "~> 0.1.0"
    }
  }
}

provider "powersync" {
  # admin_token is picked up from the PS_PAT_TOKEN environment variable (recommended).
  # It can also be inlined — less secure, but if inlined via a variable the value
  # itself can still be passed securely through Terraform's own env vars
  # (e.g. TF_VAR_admin_token).
  # admin_token = var.admin_token
}
```

A complete end-to-end example — organization lookup, project, and a fully-configured instance — lives in the [`powersync_instance` resource docs](resources/instance.md).

## Quickstart

```hcl
terraform {
  required_providers {
    powersync = {
      source  = "powersync-ja/powersync"
      version = "~> 0.1.0"
    }
  }
}

provider "powersync" {}

data "powersync_organization" "main" {
  id = "64b3f8e1a2c4d5e6f7080912" # from your dashboard URL
}

resource "powersync_project" "main" {
  org_id = data.powersync_organization.main.id
  name   = "My Project"
  region = "eu"
}
```

```sh
export PS_PAT_TOKEN="jpt_..."
terraform init
terraform apply
```

For a walkthrough that goes all the way to a running instance synced from a Supabase database, see the [Getting Started guide](guides/getting-started.md).
